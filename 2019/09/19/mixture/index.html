    • \d 数字：[0-9]

    • \D 非数字： [^0-9]

    • \w 单词字符：[a-zA-Z_0-9]

    • \W 非单词字符：[^\w]

      算法

    • [a-d[m-p]] a到 d 或 m 到 p：[a-dm-p]（并集）

    • [a-z&&[def]] d、e 或 f（交集） // 类似减法

      次数类

    • X? X，零次或一次 （不能是多次）

    • X* X，零次或多次 （除了0次）

    • X+ X，一次或多次 （拒绝0次）

    • X{n} 恰好 n 次

    • X{n,} 至少 n 次

    • X{n,m} 至少 n 次，但不超过 m 次
      （非捕获啥的有点蒙）

    • Python的性质

    1. 优雅（简单易学）
    2. 可移植性强

      reason：Python语言写的程序不需要编译成二进制代码,可以直接从源代码运行。在计算机内部，Python解释器把源代码转换成称为字节码的中间形式，然后再把它翻译成计算机使用的机器语言并运行。不再需要担心如何编译程序，如何确保连接转载正确的库等等，所有这一切使得使用Python更加简单。

    3. 丰富的库（爬虫的好帮手）

      正则表达式、文档生成、单元测试、线程、数据库、网页浏览器、CGI、FTP、电子邮件、XML、XML-RPC、HTML、WAV文件、密码系统、GUI（图形用户界面）、Tk和其他与系统有关的操作.

    4. 可扩展性：部分程序用C/C++写 （v提升）
    5. 可嵌入性：把Python嵌入你的C/C++程序，从而提供脚本功能。

    maven

    学习的原因：项目构建过程包括【清理项目】→【编译项目】→【测试项目】→【生成测试报告】→【打包项目】→【部署项目】

    理想的项目构建是高度自动化，跨平台，可重用的组件，标准化的，使用 maven就可以帮我们完成上述所说的项目构建过程。

    本地仓库统一管理jar包，使jar包与项目分离，减轻项目体积
    多个项目的时候就能体会到jar包的好处

    优点：

    jar包管理 也称为依赖管理
    Maven由Java语言开发，可跨平台使用

    --- Maven将大型项目按照模块拆分成若干块，不同团队来维护，之间用远程仓库 中小型项目：一般分为**web、service、dao**层 Dao层叫做数据访问层，data access object Service层用**接口**来定义

    用终端跑起来
    Cd到目录，之后输入 mvn tomcat:run
    编译过后src目录层会多出target
    Tip：如果本地仓库没有jar包，就会自动到互联网上下载

    如果用maven，需要打包src文件和pom文件

    Maven的三个仓库
    本地仓库：自己维护
    远程仓库：公司维护 私服
    中央仓库：第三方团队维护 国内一般使用阿里云提供的（到mirror里面去配置，如果不配置，就会到maven官方网站去下载，不挂代理的话，会非常慢）（是啥？）

    坐标一般是公司提供的，坐标是groupID、artifactid, Version

    常用命令：

    --- Mvn **clean** 是清除编译后的文件 Mvn **compile** 编译：只编译main目录下的代码，不编译test下的 Mvn **test-compile** 编译test下的代码 **Test** 是运行test下的代码 **Package** 打包,包名是由artifactID，version，packaging（war、jar包）来的 **Install** 是上传到本地仓库 **Tomcat:run** 一键构建项目

    sed

    p 显示
d 删除
a 添加
c 替换
w 写入
i 插入

    1/ 删除：d命令

    $ sed '2d' test.txt —–删除example文件的第二行。

    $ sed '2,$d' test.txt —–删除example文件的第二行到末尾所有行。

    $ sed **'$d'** test.txt —–删除example文件的最后一行。

    $ sed **'/test/'d** test.txt —–删除example文件所有包含test的行。

    2/ 替换：s命令

    sed 's/test/mytest/g' test.txt

    在整行范围内把test替换为mytest。如果没有g标记，则只有每行第一个匹配的test被替换成mytest。

    /sr`v/testesd# sed -n's/^test/mytest/p' test.txt`

    (-n)选项和p标志一起使用表示只打印那些发生替换的行。也就是说，如果某一行开头的test被替换成mytest，就打印它。
    并没有办法使用！！！

    sed 's/^192.168.0.1/&localhost/' test.txt

    &符号表示追加一个串到找到的串后。所有以192.168.0.1开头的行都会被替换成它自已加 localhost，变成192.168.0.1localhost。（想要在某个单词后加东西的时候使用）

    sed -n's/loveable/\1rs/p' test.txt

    ove被标记为1，所有loveable会被替换成lovers，而且替换的行会被打印出来。（首先语句的意思不太明白，其次存在-n就会报错）

    换新的分隔符，不感兴趣！

    3/ 从文件读入：r命令

    sed  '/test/rfile'  test.txt

    —–file里的内容被读进来，显示在与test匹配的行下面，如果匹配多行，则file的内容将显示在所有匹配行的下面。（不太明白其中的意思！！！）

    Lucene介绍

    是一款高性能的、可扩展的，纯java语言编写的一个开放源代码的信息检索（IR）工具库。它适合几乎任何需要全文本搜索(特别是跨平台）的应用程序。但它不是一个完整的全文检索引擎，而是一个全文检索引擎的架构，提供了完整的查询引擎和索引引擎，部分文本分析引擎（英文与德文两种西方语言）。Lucene的目的是为软件开发人员提供一个简单易用的工具包，以方便的在目标系统中实现全文检索的功能，或者是以此为基础建立起完整的全文检索引擎。
